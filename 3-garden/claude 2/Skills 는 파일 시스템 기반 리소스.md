### Skills는 파일 시스템 기반 리소스다.

skill은 단순한 프롬프트 문장으로 존재하는게 아님. 
__로컬 파일 시스템 안의 폴더 구조와 파일세트임__

각 skill은 하나의 __독립된 리소스 패키지__. 모델은 __이 폴더를 탐색하고 로드__ 하여 사용

```
skills/
 └─ figma-to-react/
     ├─ SKILL.md               # 이 스킬의 설명서(가이드라인, 규칙)
     ├─ examples/
     │   ├─ button.json
     │   └─ card.json
     ├─ schema/
     │   └─ component.schema.json
     └─ scripts/
         └─ transform.ts
```

- SKILL.md를 읽어 규칙을 파악
- __필요할 때__ examples/ 안의 JSON 등을 참조

- __단일 프롬프트에 모든 정보를 한번에 넣는 대신__ VS 모델이 필요할 떄마다 파일을 탐색하여 점진적으로 컨텍스트를 확보하는 구조

| 항목           | Skill을 직접 실행 (`run skill ...`)                        | 프롬프트로 SKILL.md 내용을 붙여넣음     |
| ------------ | ----------------------------------------------------- | --------------------------- |
| **구조적 인식**   | 파일시스템 기반 리소스로 인식 → 내부 파일/예시/코드 참조 가능                  | 단순 텍스트로 인식 → “문서 내용”으로만 읽음  |
| **컨텍스트 관리**  | Claude가 “skill manifest”로 등록하고 필요시 참조                 | 단일 세션 내 ephemeral(휘발성) 컨텍스트 |
| **명령 실행 흐름** | Skill 내부의 절차적 정의를 따라 단계별 실행                           | 자연어 해석을 통해 모델이 자체적으로 “흉내”   |
| **일관성**      | 코드화된 절차를 항상 동일하게 실행                                   | 프롬프트에 따라 유추 방식, 해석이 달라짐     |
| **조합 가능성**   | 다른 Skill과 연동 가능 (ex. `figma-to-react` → `lint-react`) | 매번 수동 연결 필요                 |
| **메모리성**     | Skill은 파일 시스템에 존재하므로 장기 재사용 가능                        | 세션 종료 시 사라짐                 |
