#### **1. TS를 쓰는 회사에서 깃허브로 평가하는 기준**

##### **(1) any의 사용 여부**

- **any의 문제점:**
    - 타입 안정성을 포기.
    - 잘못된 데이터 흐름을 사전에 방지하지 못함.
- **대안:**
    - **unknown:** 타입 안정성을 유지하면서도 유연하게 데이터 처리 가능.
    - **Generic Type:** 여러 타입을 안전하게 처리 가능.
- **사용 원칙:**
    - "급할 때만 any를 쓰겠다"는 주관적인 기준은 결국 남발로 이어질 가능성이 큼.
    - **아예 쓰지 않겠다는 규칙**을 세우고 학습을 통해 극복해야 함.

##### **(2) Props로 `Dispatch<SetStateAction<T>>`를 넘기는 문제**

- **문제:**
    - 부모가 `useState`를 사용하는 구조를 암묵적으로 강요.
    - 함수의 재사용성과 캡슐화가 저해될 가능성이 높음.
- **원칙:**
    - 함수는 자신이 수행하는 역할에만 집중.
    - 상태 관리와 로직을 분리하여 설계.
