## 코테

그냥 나는 수능 문제 같다고 생각한다. 이 문제가 어떤 개념을 활용해야 하는지 빠르게 파악하여 제한 시간 내에 풀어야 하기 때문이다. 평상시에 준비를 잘 해둬야 당황하지 않고 풀 수 있다. 딱히 방법이 없다. 자료구조는 스택, 큐, 힙, 트리 탐색이나 최단경로쪽으로는 DFS, BFS, 이진 탐색, 정렬은 기본적인 정렬 방법들.. 정도 알면 되지 않을까 이외에 문제가 나온다면 평상시 실력으로 풀거나 그냥 틀리고 다음 회사로 가도 된다고 생각한다.
개인적으로 이런거에 시간쏟는건 알고리즘 즐기는 분 아니면 힘들고 실무에 크게 도움 안된다고 생각한다.

## 과제

아무것도 쓰지 않고 뽀모도로가 되는 투두리스트, 무한스크롤로 상품리스트가 쭉 나오고 정렬,필터 기능이 있음. 이 과제들은 전부 테스트 코드 권장사항. 3일~1주일 사이
- 기능 구현은 기본
- 코드 품질
	- 코드 가독성
		- 불필요한 중복 코드가 있는지
		- 변수와 함수명의 의미
	- 코드 구조
		- (극단적으로) 한 파일에 모든 코드를 몰아넣지 않고 기능 별로 분리했는가
		- (더 나아가서) SRP가 잘 되어있는가
		- 혹은 적절한 추상화 혹은 모듈화를 통해 중복을 제거하고 재사용성을 높였는가
	- 유지보수성
		- 변경하기 쉬운 코드인가
		- 예컨대 데이터와 UI의 분리가 잘 되어있는가 
- 테스트 코드 작성여부
	- 상태관리에 대한 테스트 구현 혹은 cypress로 동작 테스트
- 상태 관리
	- 예측 가능성과 유지보수성이 핵심
	- 전역 변수 당연히 X
	- 리덕스와 같이 단방향 데이터 흐름 혹은 pub/sub 패턴 등
- 렌더링
	- 리액트 스럽게 상태 변경시 리렌더가 이해하기 쉬운 코드.
- 컴포넌트 구조 설계
	- 관심사 분리를 통한 분리나 모듈화가 잘 되어있는가
		- App.js에 모든 코드가 몰려있는 안티패턴
	- 단일책임원칙
		- UI(tsx)와 데이터(hook or redux와 같은 상태 관리)의 분리
		- UI가 상태관리에 의존적인가
- 안티패턴
	- id, class로 DOM에 의존한 로직 작성
	- 명령형패턴

카카오페이나 토스 증권, 업비트와 같이 실시간 성이 중요한 서비스
웹소켓을 이용하여 실시간성 서비스 구축경험

next.js app router를 활용하여 상품 상세 페이지(당근..)
- 상품 상세 API
- 해당 유저의 상품 판매 리스트 API
- 인기 매물 API
- 페이지라우터 처럼 구현했는지 app router처럼 구현했는지
	- 서버 컴포넌트를 통해 먼저 처리된 곳부터 렌더링
		- 기존엔 SSR을 하더라도 한꺼번에 HTML 보내야함.
	- 그래서 나머지 HTML 추가 전송
	- 클라는 받은 HTML을 조합해서 점진적 UI를 완성
