   - 동일한 해시값이 생성되는 이슈
     1. 체이닝
        - 충돌 발생시 연결리스트나 트리를 이용하여 데이터 저장 
        - 장점: 충돌이 많아도 동적으로 저장 가능하여 메모리 효율적 사용 가능
        - 단점: 충돌 심한 경우 검색 속도가 O(n)
     2. 오픈 어드레싱
        - 해시맵의 다른 빈버킷을 탐색하여 저장 
           - 탐색 방식
              - 선형 탐색: 고정된 간격(+1)으로 버킷 순차적 탐색
              - 이중 해싱: 2개의 다른 해시 함수 사용
           - 장점: 별도의 자료구조 필요하지 않음
           - 단점: 클러스터링(데이터 밀집 현상) 발생하여 성능 저하 가능성
     3. 이중 해시
        - 추가 해시함수 사용하여 충돌시 대체 위치
     4. 완전 해시
        - 해시 충돌 발생하지 않도록 설계된 해시. ㅏㅁ수 사용
     5. 동적 크기 조정
        - 충돌 비번하거나 해시맵 용량 초과시 해시맵의 크기 증가. 후에 모든 데이터 다시 해싱 하여 저장
        - 장점: 충돌 확률 줄이고 검색 성능 유지
        - 단점: 해싱, 재배치 과정에서 추가 비용 발생

```js
function fn(v) { 
  v = {name: 'react'}

var k = {name: 'next'}

```
- 해설 
	- 객체를 함수에 전달할 때 해당 객체의 참조가 복사된다.

	- 함수 내부에서 매개변수에 새로운 값을 할당시, 해당 참조가 새로운 값을 가리키도록 바뀐다. 하지만 . 이변경은 함수 외부에는 영향을 미치지 않는다. 
	- 중요 포인트: 함수 내부에서 v에 새로운 객체를 할당한 것은 v의 참조를 바꾼것이다. k가 참조하는 . 걸바꾼게 아니다.

3. arrow function vs function
   ```js
   function abc() { 
     console.log(this); 
   } 
   
   abc(); 
   
   var obj = { 
     fn: function() { console.log(this) },
     arrowFn: () => { console.log(this) } 
   } 
   obj.fn(); 
   obj.arrowFn();
```

-  해설
- abc(): 일반 함수 호출의 this는 전역객체를 참조
- obj.fn(): this는 객체를 가리킨다. 객체의 메서드 호출은 호출한 객체가 this로 바인딩
- obj.arrowFn(): 화살표 함수는 자신의 this를 렉시컬하게 상속받는다. 즉, arrowFn이 ==정의된 위치의 this==를 따른다.
	- arrowFn은 obj객체 내부에서 정의되었지만, ==정의 당시의 this는 상위스코프 this==다.
	  
> 화살표 함수의 this는 일반 함수처럼 호출 방식에 의해 동적으로 바뀌지 않는다. 화살표 함수가 정의된 위치의 상위 스코프 this를 그대로 가져온다. 
> 렉시컬 스코핑: 코드가 작성된 문맥에 따라 변수를 참조
