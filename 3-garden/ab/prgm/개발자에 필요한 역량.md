#### **1. JS API 외우기 vs 원리 탐구**

- **단순 암기보다 원리를 탐구:**
    - JS API를 많이 외우는 것 자체는 중요하지 않음.
    - **“왜 이 기술이 존재하는지”에 대한 원리와 배경을 이해하는 것이 더 중요.**
    - API 사용법은 필요할 때 문서를 참고하면 되지만, 기술의 존재 이유는 스스로 고민하고 탐구해야 함.

#### **2. 하드스킬 vs 소프트스킬**
- **하드스킬:**
    - 혼자서도 충분히 기본을 익힐 수 있음.
- **소프트스킬:**
    - **“집단 내”에서만 배울 수 있는 능력.**
    - 원활한 커뮤니케이션과 팀워크가 필수적.
    - 개발자의 소프트스킬 예시:
        - **팀 미션 달성:**
            - 원활한 소통을 통해 팀 목표를 효과적으로 이루는 능력.
        - **의사결정 근거 설명:**
            - 논리적이고 명확하게 자신의 선택을 동료(개발자, 비개발자)에게 설명할 수 있어야 함.
            - **공감 가능한 가치**를 찾아내는 능력:
                - 개발자만 이해할 수 있는 가치(빌드 시간 단축)보다는, 모두가 이해할 수 있는 가치(시간 절약, 비용 절감)에 초점.

#### **1. 하드스킬 vs 소프트스킬**

- **소프트스킬을 더 중시:**
    - 단순히 코딩을 잘 하는 사람이 아니라, **동료들과 논의하며 비즈니스 임팩트를 만들어내는 사람**을 원함.
    - 커뮤니케이션 역량이 무엇보다 중요:
        - 일정이 틀어질 때 이를 **즉각적으로 공유**하는 능력이 더 중요.
        - 팀 내 원활한 소통이 좋은 동료로 평가받는 핵심.
    - **동료들의 기대:**
        - 동료들은 **FE 코드 퀄리티를 평가할 수 없음.**
        - 대신, "우리가 원하는 것을 만들어주는 사람", "소통이 잘 되는 사람"을 원함.
        - 주니어 개발자들이 종종 간과하는 부분임.

---

#### **2. PO/PD의 입장에서 생각하기**

- **관점을 바꿔라:**
    - 내가 PO라면 어떤 개발자를 원할까?
    - 내가 PD라면 어떤 개발자를 원할까?
    - 동료 입장에서 한 번 생각해 보면, 커뮤니케이션과 협업의 중요성을 쉽게 이해할 수 있음.

---

#### **3. 하드스킬의 기준은?**

- **어느 정도여야 하나?**
    - 만들고 싶은 것을 **만들 수 있는 정도**면 충분.
    - 모든 것을 미리 아는 것은 불가능하며, 필요한 기술은 **그때그때 배우고 적용**하면 됨.
    - 중요한 것은 **리서치하고 학습하여 활용하는 능력.**
- **암기보다 이해가 중요:**
    - 기술을 단순 암기하는 것은 비효율적.
    - 새로 나오는 기술은 기존 기술의 응용이므로, **기존 기술에 대한 깊은 이해**가 학습 효율을 높임.
    - **학습 곡선:**
        - Linear하게 암기하지 말고, Exponential하게 이해를 기반으로 확장하라.

---

#### **4. 핵심 요약**

- **소프트스킬 중시:**
    - 동료와의 소통, 일정 공유, 협업 능력이 가장 중요.
    - 기술력보다 비즈니스 목표를 이해하고 이를 달성하려는 자세.
- **하드스킬은 기본적인 제작 능력과 학습 능력:**
    - 모르는 것을 필요에 따라 배우고 활용할 수 있는 능력이 더 중요.
    - 암기가 아니라 **기술 이해**를 통해 학습 효율을 높이는 것이 관건.




#### **3. 학습 태도와 납득의 기준**

##### **(1) 성장 속도가 빠른 사람의 특징**

- 납득의 기준이 높음:
    - "아 그렇구나"로 끝내지 않고 계속 의문을 파고듦.
- **드릴다운 방식:**
    - "비동기 프로그래밍은 HTTP 요청을 제어하는 것"이라는 설명이 있다면:
        - **비동기가 뭔데?**
        - **비동기에서 무엇을 제어하는데?**
        - **HTTP란 무엇인가?**
        - 의문에서 의문으로 연결하며 학습의 깊이를 확장.

##### **(2) 개인적인 학습 방식 공유**

- **멘토의 접근법:**
    - 타인의 설명을 무조건 믿지 않음.
    - 상대방의 말에서 논리적 허점을 찾고 스스로 검증.
    - 지식의 부족함을 느끼면 집에 와서 추가 조사.
- **학습 원칙:**
    - "세상에 정답은 없다"는 태도를 바탕으로 스스로 탐구.
    - 강하게 주장하는 정보는 경계하고, 직접 확인과 검증.
