- **질문:**
    - 이전 프로젝트에서 받은 인프라 관련 피드백을 완전히 이해하지 못함.
    - 개발 공부와 인프라 이해의 우선순위 설정이 필요함.
- **피드백 요약:**
    - **개선사항:**
        - 서로 다른 VPC 통합 → 네트워킹 비용 절감.
			- aws를 둘 다 사용하지만 서로다른 vpc 요청은 네트워킹 비용 발생
			- vpc 피어링으로 vpc를 묶는다 (CIDR)
        - EC2 대신 Amplify로 정적 웹 호스팅 전환 → 과금 절감 및 배포 관리 간소화.
	        - EC2 + docker 배포 파이프라인 복잡 -> EC2 사용하기에 과금 발생
	        - 도메인 분리가 필요한 기능 추가시 CName 관리와 배포관리 용이함(개발과 프로덕트 환경 분리)
        - 동일 VPC로 통합하여 도메인 관리 비용 절감.
- **조언:**
    - **신입 개발자에게 기대 수준:**
        - 인프라는 신입에게 기대치가 높지 않지만, 스타트업 환경에서는 필요할 수도 있음.
    - **알아두면 좋은 지식:**
        - **AWS 기본 지식:**
            - S3, CloudFront, Lambda Edge 등 주요 서비스 이해.
        - **네트워크 개념:**
            - TCP/IP, Inbound/Outbound, VPC 등 기본적인 네트워크 구조와 필요성을 이해.
        - AWS 제품 전반에 대한 개괄적인 이해.
    - **공부 방법:**
        - 가능한 많이 실습해보는 것이 중요.
        - 학생이라면 AWS Educate 프로그램 활용 가능 (비용 절약).
