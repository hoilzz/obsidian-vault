## 1. so-so.dev

저는 디자인 시스템, 훌륭한 웹 서비스, 아름다운 것들을 사랑하는 UX 엔지니어입니다.  
같은 자원을 활용해 어떻게 하면 최대의 영향을 낼 수 있을지 항상 고민하지만, 때로는 비효율적인 선택이 필요하다고도 믿습니다.  
저는 변화에 유연한 디자인, 시스템, 테스트, 자동화에 투자하며, 이러한 요소들이 제품 개선과 함께 유기적으로 작동할 수 있도록 노력합니다.

저는 제 경험과 지식을 공유하는 것을 좋아합니다.  
지식을 공유할 수 있을 때 비로소 그것이 내 것이 된다고 믿으며, 이러한 공유의 선순환에 기여하고 싶습니다.

        - Toss 계열사에서 사용되는 새로운 PC 디자인 시스템을 설계하고 개발하였습니다.

    - **제품 컴포넌트 추상화 수준 정의 및 디자인 시스템 책임 범위 설정**
        - 제품을 구성하는 컴포넌트를 네 가지 레이어(도메인 컴포넌트 조합 / 도메인 컴포넌트 / 제품 시스템 / 디자인 시스템)로 구분하고 각 레이어의 책임 범위를 정의하였습니다.

    - **안정성과 신뢰성을 위한 테스트 도입**
    - **새로운 시스템으로의 마이그레이션을 위한 코드 전환(codeshift) 및 생태계 정리**
    - **더 나은 UX를 위한 제안 및 설계 참여**

#### flex

**2021.02 ~ 2022.11**

- **포지션**: 디자인 플랫폼 및 Core Squad
- **프로젝트**: Design System, flex2.0 개편, Core HR 관련 서비스
    - **Design Platform**
        - **flex Design System 3.0(linear) 설계 및 개발**
            - Design System 2.0에서 얻은 교훈을 바탕으로 새로운 디자인 시스템을 구축하였습니다.
            - 디자이너와 개발자의 협업을 돕고, 제품의 유연성, 심미성, 접근성을 개선하였습니다.
            - 동작의 일관성을 보장하면서 표현의 제약을 줄이는 디자인 시스템을 설계하였습니다.
            - 다양한 컴포넌트 사용을 효율적으로 지원하기 위해 추상화된 컴포넌트를 정의하였습니다.
            - 동작의 일관성을 보장하기 위해 헤드리스 컴포넌트를 구성하였습니다.
            - 확장성(linear)에 중점을 둔 시스템과 생산성(linear-extension)에 중점을 둔 시스템으로 패키지를 분리하였습니다.
            - **시스템 세부 구성**
                - Linear의 모듈을 사용해 제품에서 자주 사용하는 UI 패턴을 Linear-extension의 모듈로 제공합니다.
                - 컴포넌트 시나리오 기반으로 테스트를 구성하였습니다.
        - **디자이너와 개발자의 생산성을 높이기 위한 도구 기획 및 개발**
            - Figma에서 관리되는 디자인 리소스를 코드와 동기화하는 스크립트를 작성하여 리소스 변경 시 발생하는 커뮤니케이션 비용을 줄였습니다.
            - Skeleton Code Generator Figma 플러그인을 개발해 개발자 생산성을 높였습니다.
    - **flex2.0 개편**
        - 다양한 고객 규모와 HR 정책을 지원하기 위해 제품을 개편하였습니다.
        - 작업의 기반과 우선순위를 정리하며, 대규모 변경 사항을 빠르고 안정적으로 구현할 수 있도록 로드맵을 수립하였습니다.
        - **2.0 프로젝트 설계 및 기본 패키지 생성**
            - React Query를 사용할 때 queryKey를 고유하게 설정해야 하는 부담을 덜기 위해 API 요청 정보를 기반으로 queryKey를 자동 생성하는 Hook 패키지를 개발하였습니다.
            - stitches.js, react-hook-form과 같은 flex1.0에서 사용하지 않았던 라이브러리를 도입하고, 팀 내에 사용자 가이드를 전파하였습니다.
            - Ant Design 기반의 Design System 2.0을 개발하고 유지 관리하였습니다.
            - Framer 코드 컴포넌트 구성을 개선하여 디자인 및 개발 단계에서 하나의 구현으로 사용할 수 있도록 하였습니다.
        - **유저 프로필, 멤버 추가/초대**
            - Uncontrolled와 Controlled 방식의 조합으로 복잡한 양식 설계 및 개발.
            - 권한 처리를 간소화하도록 설계.
		- 
    - **Project Performance TF**
        - TF에서 패키지 번들 설정을 개선하여 제품 속도를 개선하였고, 번들 크기를 총 2.52MB에서 1.84MB로 줄였습니다.
    - **Core HR**
        - Ag-Grid를 활용해 대규모 데이터를 편리하게 수정할 수 있도록 Editable Table의 공통 모듈화를 진행하였습니다.
        - HR/계약 정보의 대량 수정 기능을 이를 기반으로 개발하였습니다.
        - 회사별로 멤버 초대 프로세스를 맞춤화할 수 없는 문제를 해결하기 위해 멤버 초대 템플릿을 관리할 수 있는 온보딩 기능을 개선하였습니다.
