### 이력서 피드백

- 핵심역량
	- 설계, 개발, 배포, 운영까지 경험 보유
		- 이 경험이 강점이 될텐데 이 시간 동안 겪은 시행착오나 기술결정에 대한 내용이 블로그에 담기면 좋을 것 같다.
		- 혹은 깃허브에서 이와 관련된 내용을 참고할 수 있으면 더 좋다.
	- 대규모 데이터 처리, 트래픽 관리 및 최적화
		- 프론트엔드 시장이 next js로 옮겨가면서 서버 및 인프라 역량도 중요시 되고 있다. (중니어~시니어 레벨에서) 
		- 이 경험에 대한 얘기도 블로그를 쓰면 아주 좋을 것같다
		- 스트레스 테스트 툴을 사용하여 TPS 개선기 등..
			- SSR, SSG 적절한 활용, 서버 컴포넌트 적극 활용, 캐싱(API 응답, RSC payload), 네트워크 최적화 (CDN, HTTP 캐싱 keep-alive)
	- UI/UX 개발
		- 내용이 추상적임
		- 디자인 시스템 혹은 재사용성이 높은 컴포넌트 구조 설계와 같은 내용이 들어가면 좋을 것 같다
	- 문제해결능력
		- 100만큼 소요되던 퍼포먼스가 XX%로 감소. 이와 같은 내용이 많던데 이런 내용을 적으면 더 수치화되서 좋을 것 같다.


- 경력
	- 디피플래닝
		-  리뷰 필터기능 2s -> 200ms 단축
			- FE단에서 감소한건지.. 혹은 API단에서 감소한건지 잘 모르겠음
			- 하위 항목에 근거가 나와야되는데 부족함.
			- 초기 고객이 요청을 할때 DB, API에서 정보를 가져오는 것이 아닌 페이지 로딩때 데이터 호출
				- 이게 무슨말인지 모르겠음
		- 사용자 키워드 검색 대기 시간 15분에서 3분으로 3초로 감소
			- 이게 무슨말인지 모르겠음
