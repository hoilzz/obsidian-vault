## 2. 지원자의 현재 상황과 목표 체크
- 안녕하세요. 저는 피부과 EMR 차트 프로그램 개발 관련 회사로 입사한지 2개월된 프론트엔드 신입 개발자입니다.(기술 스택은 리액트와 타입스크립트 사용합니다.)  
  
그동안 부트캠프를 하면서 진행했던 프로젝트가 대부분이다보니, 너무 시간에 쫒기는 구현만 해왔던 것 같습니다.  
그러다보니 깊이 있는 학습이나, 공부 방법 및 방향성에 대해 아직 확신이 없고, 많이 부족하다고 느끼고 있습니다.  
냉정하게 같은 1년차 개발자들에 비해서도 뒤쳐진다고 느끼고 있습니다. 최근에는 회사 도메인 파악과 코드 분석 위주로 학습을 하고 있었고, 실무 투입도 되고 있습니다.  
기존 중복 코드 공통 컴포넌트 작업을 했었구요. 역시나 실무를 하다보니 부트캠프때의 습관이 그대로 드러나는 것 같습니다. 마감기한에 쫓기면서 AI(커서)를 활용하며 빠르게 구현을 해야하는 압박감으로 진행하고 있습니다.  
요즘 생각이 사이드 프로젝트를 하나 따로 진행하면서 깊이 있는 학습을 해야하나? 라는 고민이 드는 것 같습니다.   
  
고민은 아래와 같습니다!  
  
1. 앞으로의 공부 방향성과 공부 습관을 어떻게 잡고 성장해나가면 좋을지.  
   -  역량으로 회사에 변화를 만들어내보내는 경험. 거창하지만 내자신 -> 팀 -> 회사
   - 기술의 수준이 아닌 기술의 방향. 회사가 당면한 문제를 이해 -> 필요한 일을 정의 -> 우선순위를 수립하여 ㄱㄱ
   - 기술은 수단이지 목적이 아니다. 필요한 결과를 만들어내ㅏ.
2. 회사에서 어떻게 성장을 해나가면 좋을지.(개발 조직 문화가 좋은 편은 아닙니다.)  
	1. 프 4, 백 4, 기획 4: 프론트는 동기까지 2명, 2년, 3년차
	2. 구현만 되면 된다. 코드 퀄리티 고민 없음. 리뷰가 없음. 
	3. PR 문화를 도입. 
3. 만 1년차때도 이직을 해도 괜찮은지? 괜찮다면 1년이라는 기간동안 어떻게 방향성을 잡아가면 좋을지  
4. 근무 시간 외에 퇴근 후나 주말에는 어떤 방향으로 학습해 나가면 좋을지  
5. 인강이나 도서 학습을 할 때 올바른 공부 방법  
6. 코드를 바로바로 작성해 나가는게 아니라 고민하는 시간이 길어지게 되면서 속도도 안나게 되는 것 같은데, 어떻게 개선하면 좋을지  
(여기서 말씀드린 고민은 논리적인 고민이라기 보다는, 코드를 어떻게 짜야할 지에 대한 비중이 더 큰 것 같습니다.)  
6. 부끄럽지만 멘토님처럼 네카라쿠배 급의 기업을 목표로 한다면 지금부터 어떻게 방향을 잡고 성장해 나가야 할 지  
7. AI에 대체되지 않는 개발자로 살아남으려면 지금부터 어떻게 방향성을 잡고 나아가야 할 지
   - expert generalist
