- Next.js 마이그레이션
	- 어떤 페이지를 작업했고, 작업 인원과 제한적인 환경을 잘 작성하신 것 같다. 
	- 작업 리스트
		- _가장 중요한 파트. 여기서 가장 주요하게 보는건_
			- 어떤 문제를 해결하기위해 무엇을 했고 성과는 무엇인지(Why가 중요)
			- 혹은 어떻게 했고 이것으로 어떤 성과가 있었는지(How + Why)
			- anti pattern: XXX 도입, XXX 구현
			- 물론 모든 내용에 대해 Why를 쓰기 어려울 수 있습니다. 하지만 성과나 이유에 대해 작성하지 않으면 다른 지원자와 차별성이 없습니다. 
		- 일단 평가자 입장에서 보기 편하게 
			- 어떤 기능 구현을 했는지 (회원가입/로그인)
				- 기능을 먼저 작성하여 평가자에게 배경지식을 조금 주면 아래 기술적인 내용을 이해하는데 도움을 줄 수 있어요
			- 그 기능 구현동안 어떤 기술적인 도입을 했는지 (이외)
		- 일단 Next.js를 통해 어떤 성과를 달성했는지 혹은 기존에 어떤 문제 때문에 도입했는지 그리고 그 문제를 해결한 성과는 무엇인지에 대한 내용이 가장 궁금했지만 그 내용이 없네요.
			- 물론 next.js가 업계 표준이니까 사용했을 수 있지만 이 기술을 도입하여 어떤 문제를 해결하여 어떤 성과가 있었는지를 고민하여 코드를 통해 실행하는 것이 차별성을 보여줄 수 있습니다.
			- ex. next.js 캐싱, 앱 라우터, 혹은 흔한 SSR로 SEO 개선 등..
			- 공홈문서를 읽거나 컨퍼런스 혹은 링크드인의 네임드 개발자들의 레주메나 글들을 참고하여 실행으로 옮기고 이력서에 추가하면 좋을 것 같아요.
		- 세션 관리, 주요한 라이브러리 (상태 관리, 데이터 패칭, 스타일) 선정, 배포 등 프로젝트 초기 설계에 주도적으로 참여 했습니다.
			- 많은 내용이 담겨있네요 ㅎㅎ
			- 세션 관리는 '어떻게' 그리고 왜 그 방법으로 구현했는지가 중요할 것 같네요 ㅎㅎ
				- _세션을 레디스 or 쿠키_ 기술적
					- 쿠키 암호화.
				- ex. next js 기반 리버스 프록시 구조로 인증 책임 및 프로세스 통합 관리
					- (bullet 형으로 조금 더 보충 설명하자면)
					- 미들웨어에서 공통 인증 파이프라인을 구현하여 모든 API 호출에 인증 헤더 주입 및 세션 검증
		- TypeScript 도입
			- TS 도입으로 인한 효과를 같이 적어주면 좋을 것 같아요
			- 개인적으로는 단순 TS 도입이라면 기술 스택에 Typescript 키워드 추가 정도로 해도 될것같아요
		- SNS 회원가입/로그인 기능 구현
			- 단순 기능 구현은 Why, How를 쓸 수 있으면 좋지만 그렇지 못한 경우가 많습니다.
			- 이정도로 괜찮을 것 같아요. 순서만 변경 부탁드려요
		- 서드파티 결제 모듈 연결
			- 단순 연결 혹은 연결 후에 이것을 통해 추가로 개선한 작업이 있다면 작성해주세요
		- 애널리틱스도 위와 동일합니다
		- 기존 로그아웃 UX에 문제점을 파악 기술적인 솔루션 주도적으로 제시
			- 평가자입장에서 _어떤 문제가 있었는지_
