## 3. 이력서 리뷰


- 지금 이력서를 봤을 때는 멘토님께서 언급하신 잘못된 접근인 '나 이거 개발했음'의 나열처럼 보이는 것 같습니다. 문제 해결 과정을 이력서에 어떤 식으로 녹여낼 수 있을지 감을 잡기가 어렵습니다.  
- '더 나은 유저 경험을 위해 고민하는 개발자'라고 캐릭터를 잡았지만, 다소 모호하다고 생각이 듭니다. 좁혀서 캐릭터를 잡을 필요가 있을지 여쭤보고 싶습니다.  
- 유저가 서비스에 어떻게 접근하고 어떻게 이용할지 고민하며 개선해나가는 것이 제 강점이라고 생각합니다. 해당 역량을 기술 역량과 엮어 어떤 식으로 정량화/수치화시켜서 표현해야 할 지 모르겠습니다.  
- 현재 이력서의 프로젝트명을 누르면 해당 프로젝트에 대한 상세한 설명이 뜨게 되는데 해당 내용이 필요할 지, 적절한 연결 방식일 지 궁금합니다.
