- 가비지 컬렉터 설명해주세요
	- 더 이상 참조되지 않는 객체를 자동으로 정리
	- 가비지 컬렉터는 브라우저에서 어떻게 동작하는지 아는가?
		- Mark and Sweep 알고리즘 주로 사용
			- Mark(표시): root 객체(window, document)부터 탐색 시작
				- 루트 객체에서 접근 가능한 모든 객체는 현재 사용중인 객체로 표시 
			- Sweep(제거): 도달 가능한 객체로 표시되지 않은 객체를 가비지(더이상 사용되지 않는 메모리)로 간주하고 제거
				- 이렇게 해제된 메모리를 재사용
	- 언제 동작?
		- 메모리 부족
		- 메모리 사용 패턴에 따라 주기적으로
	- 가비지 컬렉터가 없으면 어떻게 될까?
		- 메모리 누수
			- 사용하지 않는 객체들이 계속 메모리 점유하여 메모리 leak 발생.
		- 개발자가 직접 메모리 관리해야하는데 실수 발생할 여지 크다. 
		- 성능저하
			- 브라우저 탭이나 애플리케이션 멈춤 현상
			- 
	- 메모리 누수 겪어본적 있는가?
	- 없다면 언제 발생할 수 있을까?
		- 전역 변수 사용: window 객체에 붙어 있어 GC가 수집 불가
		- 이벤트 리스너 제거하지 않음
