#### 삼성 Rubicon

- _클래스 설계_
	- 단순 설계는 차별성이 없다. 
	- 설계시 어떤 패턴으로 작성되었고 이 패턴을 사용해서 어떤 효과가 있었는지 작성하면 더 베스트
		- 비동기 이벤트 기반 설계 혹은 전략패턴이라던가 리액트 단에서 사용한거면 훅으로 분리하면서 재사용성을 높였다던가 등..
	- 사용한게 있나요?
	- 더불어서 해당 클래스의 유닛 테스트를 활용하여 테스트 커버리지 90% 이상 달성을 통한 코드 품질 개선 도 내용이 있으면 좋다. 어쨌든 클래스 설계하면서 해당 클래스는 재사용이 많이 될텐데 테스트 코드까지 더해지면 코드 품질이 높아지는건 당연한거고 민수님이 어떤 설계를 진행할 때 이분은 테스트 코드까지 작성하여 코드 품질을 높인다는 인상을 줄 수 있다. 테스트 코드는 단순히 작성했다는 의미를 넘어서서 코드 품질을 관리하겠다는 인상을 줄 수 있다. 
	- _훅분리_
		- 이 내용은 지엽적이라 사실 크게 와닿지 않는다. 
	- _Controller 훅_
		- 뭔가 클래스 설계의 한 부분을 설명하는것 같은데 클래스가 Model View Controller 라던가 뭐 그런식으로 분리되어있다면 패턴을 적는게 나을 거 같다.
- _STT_ 최적화
	- _최적화 내용은 session stop callback을 활용한 음성인식 오나료 TTS 사이 딜레이 이슈 해결_
		- 이 내용 뿐인 거 같은데 딜레이가 as-is, to-be 가 작성되면 좋겠다
		- 이런 트러블슈팅기가 블로그로 작성되면 적절한 주제가 될 것 같다.
		- 문제가 뭐였고 문제의 원인은 뭐였고 이에 대한 해결책으로는 session stop callback을 활용하면 되었고 성과는 어떻다~ 일너거
- _TTS_ 품질 개선
	- 여기도 비슷하게.. 품질 개선은 클래스 설계일거 같은데 위와 동일한 내용으로 피드백 드릴 수 있을 것 같다.
- _커스텀 마크다운 지원_
	- -> 이렇게 쓰는 포맷 개인적으로 쪼금 아쉽다. 차라리 커스텀 마크다운을 지원하여 챗봇 UI 확장성 증가 라고 작성하면 어떨까?
	- 그리고 커스텀 마크다운 지원의 경우 마크다운을 지원한다는게 그냥 ## 이렇게 하면 h2로 헤딩으로 실시간 전환된다는 말일까?
	- 실시간으로 반영한다는 거면 더 성과를 드러내게 쓰자.
	- 실시간 마크다운 지원하여 유저 경험 및 가독성을 높임 
	- 근데 쓰다보니까 챗봇이 답변할 때 마크다운을 지원한다는 말 같네?
