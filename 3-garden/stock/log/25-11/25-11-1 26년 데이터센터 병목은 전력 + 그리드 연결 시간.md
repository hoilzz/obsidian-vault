과거 클라우드 시대의 병목은 서버/네트워크 였다. 현재 AI/HPC 의 병목은 전력 + 그리드 연결 시간이다. GPU는 현재 어느정도 돈으로 해결 가능하다. 전력과 시간은 돈으로 바로 해결되기 어렵다. 전력은 단순 비용이 아닌 전략적 자산이다. 그래서 시장이 바라보는 지점이 바뀌고 있다.

기존 병목은 GPU 수급이었고 현재는 전력망 연결로 바뀌었다. 바인딩된 전력 MW, IT load 전환 여부 로 확인할 수 있다.

AI 데이터센터 라도 밸류에이션이 갈리는 이유

| 단계  | 명칭                 | 시장이 확인하는 핵심 지표                                                                                                                           | 투자 관점                      |
| --- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| 1단계 | **전력 접근 가능**       | 확보된 부지 및 전력 할당량(MW)                                                                                                                      | '계획' 단계로 높은 변동성과 투기적 성격    |
| 2단계 | **바인딩 계약**         | 전력 구매 계약(PPA) 체결 여부 및 변전소 확보 [](https://giyooinfo.com/ai-datacenter-analysis-iren-wulf-apld-cifr-nbis/)​                                 | 실질적인 '옵션 가치' 발생 단계         |
| 3단계 | **연결 (Energized)** | 변전소 가동 및 쿨링 인프라 구축 완료 [](https://finviz.com/news/234070/how-can-irens-transition-into-the-ai-cloud-benefit-the-stock)​                   | **실행 프리미엄** 부여 (수익 가시성 확보) |
| 4단계 | **IT Load**        | 실제 가동 중인 랙(Rack) 밀도 및 점유율 [](https://www.digitalmarket.kr/web/board/BD_board.view.do?domainCd=2&bbsCd=1030&bbscttSeq=20250730170315802)​ | 현금흐름 기반의 안전자산 평가           |

### 1단계: 전력 접근 가능
1 단계는 병목, 가치 모두 작다. 단순 협의와 비용 조건을 논의한다. (스토리)

### 2단계: 바인딩 계약
2 단계는 가장 어려운 병목 구간이다. 여기에서는

- 변전소 신설/증설
	- 전력 설비 납기 12~24개월
	- 한번 지연되면 연쇄 지연
- 그리드 승인 & 테스트
	- 계통 안정성 테스트
	- 지역 전력망 충돌 문제
- 규제/주민/환경
	- 소음/열/토지 민원
	- 환경영향평가
	- 지방정부 승인 지연 
		- [[25-10-15 데이터 센터 빠르게 늘어나기 어려운 이유]]

그래서 2단계를 통과 못하면 AI 수요 많음, GPU 많음 에도 밸류에이션이 떨어질 수 밖에 없다.

### 3단계: 인프라 구축 및 연결(Energized)
3 단계는 전기는 들어왔기 떄문에 병목은 기업의 실행력 문제로 바뀐다. 대표 병목들은 다음과 같다.

- 인터커넥션(송전망 연결) 승인: '연결해주겠다'는 '언제 해주겠다'와 다름
- 변전소(서브스테이션) 신설/증설: 부품 리드타임, 공사, 인허가
- 그리드(계통) 증설 속도: 주변 송전망이 바쳐줘야 한다.
- 인허가/환경/토목: 대형 캠퍼스는 공정이 길어진다.
- 전력 확보 정의 차이: PPA나 할당량은 있어도 'enerzied'가 지연될 수 있음.

스토리->실행 으로 바뀌는 구간이라 중요.

### 기업별 상황체크

지금 전력을 얼마나 확보 했고 앞으로 얼마나 늘릴 수 있는지, 이 과정에서 변전소/그리드 병목과 같은 전력소 문제가 있는지 확인해보자.


| 회사       |                현재 확보/운영 가능 (MW) |                                             계약/확보 예정 (MW) | 비고                                                                                                                                       |
| -------- | ------------------------------: | --------------------------------------------------------: | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **IREN** |                     ~810 MW 운영용 |                      ~2.9 GW 그리드 연결 전력 확보 파이프라인 (대부분 텍사스) | “전력 자체”보단 **(대형 증설 시) 인입/증설 속도**가 병목이 될 가능성이 큼. 전력 포트폴리오가 크지만, _언제_ energize 되느냐가 핵심.                                                    |
| **APLD** |                        280MW 운영 | 공개된 것 없음<br>- 다만 총 600MW 계약 용량 있음(코어위브 400 + 하이퍼스케일러 200) | 문제는 **신규 AI 캠퍼스(’26~’27) 인입·건설 실행 리스크**(변전소/공기존 286MW는 이미 energize. 문제는 **신규 AI 캠퍼스(’26~’27) 인입·건설 실행 리스크**(변전소/공급망/인허가/공정)              |
| **CIFR** |                        207MW 운영 |                              Engie 풍력에서 최대 300MW 구매 가능 계약 | 300MW급 사이트는 **변전소/인터커넥션(“Phase II” 일정이 명시된 것 자체가 병목 관리 포인트)**. 텍사스 계열은 특히 **연결·증설 리드타임**이 리스크.                                           |
| **WULF** | BTC 245MW + HPC 22MW = 267MW 운영 |                         Lake Mariner 500~750MW(송전 업그레이드시) | - 수력 기반 지역 한계 + AI 증설 속도<br>- 수력기반은 계절성 + 지역 확장 한계<br>- 다만 “리스 체결 MW”가 “전력 인입 완료 MW”와 다를 수 있음 → **실제 energization 타임라인(변전소/송전/공정)**이 관건. |
| **CRWV** |                           590MW |             2.9GW 계약 완료.<br>하지만 이건 Core Scientific, ALPD. | 전력 자체를 많이 “계약”했어도, 실제로는 **3rd-party 데이터센터 개발자 powered-shell 지연** 같은 형태로 **인입/공급망 병목이 현실화**(경영진이 “시스템적 문제” 언급)<br>                        |
| **NBIS** |                           220MW |                                         26년까지 2.5GW 계약 목표 | 유럽 전력 제약으로 롤아웃이 느려지는 흐름이 있고, NBIS도 “capacity bottleneck”을 공식적으로 언급. 즉 **전력 확보(계약)보다 ‘연결/준공 속도’가 핵심 리스크**.<br>                            |

위 표를 통해 현재 확보/운영 가능 전력은 IREN이 810으로 가장 높다. 하지만 큰 계약이 MS외에는 없다. 이건 아직 더 상방이 열려있다는 뜻으로 해석할 수 있다. 
계약/확보 예정도 ALPD 1GW고 CRWV, NBIS 외에 2GW는 없다. 그럼 현재 IREN이 ALPD, CRWV, NBIS 외에 경쟁력 있는 것은 무엇일까?

- 코어위브, 네비우스는 고객이 많은데요?
	- 무조건 약점이 아님. 지금은 GPU -> 전력 공급으로 바뀌고 있음. 
	- IREN은 전력 먼저 확보/연결 해둠

#### iren 차별점

1. 전력 변전소를 직접 통제

| 회사       | 전력/변전소 통제권                                                              |
| -------- | ----------------------------------------------------------------------- |
| **IREN** | **자체 부지 + 직접 변전소 구축 비중 높음**. 즉, 통제권 높음.                                 |
| ALPD     | 캠퍼스 개발자 성격. IREN처럼 직접 운영하는게 아닌 캠퍼스 개발해서 임대(부동산)                         |
| CRWV     | 3rd-party powered-shell 의존. ALPD에 장기 임대. ALPD의 공사 일정에 따라 코어위브도 동시 지연.   |
| NBIS     | 국가(유럽 아시아 등)/지역 그리드 의존도 높음. (유럽 계통은 속도 한계..) 전력/부지는 혼합 조달               |
| CIFR     | 지역·유틸리티 제약 큼. 아이렌과 동일한 텍사스지만 새로 들어오려는 전력 대형 부하가 많이 걸려서 '절차/인프라'가 병목이 큼. |
| WULF     | 뉴욕의 전력/송전/규제 체계의 한계로 확장 자체가 어려움.                                        |

이게 왜 중요하냐면 2단계 계약에서 3단계 energize로 넘어가는 병목이 크다. 이 단계에서 누가 일정 통제권을 가지는지가 중요하다. 전력 자체가 아닌 속도가 병목이다. 그 속도를 상대적으로 가장 직접 통제하는건 IREN이다. 

2. 비트코인 채굴하면서 초고전력 운영 경험

이건 숫자로 보이기 힘들지만 중요한 차별점이라고 생각한다. 대규모 초고전력/변전/냉각/운영 경험이 있음. ALPD, NBIS는 처음부터 AI 전력이고, CRWV는 전력은 외주구조(ALPD)다. 

3. 전력 -> 매출로 바꾸는 시간차가 짧음

810MW를 바로 매출로 전환 가능한 상태. 전력 연결 -> IT load까지 거리가 짧다. 
반대로
- NBIS: 2.5GW 계약 -> connected 800~1GW (시간 필요)
- CRWV: 계약은 크지만 인도 지연 리스크가 있다.
- ALPD: 26~27 대형 캠퍼스

시장이 좋아지면 실적/가이던스로 반응하고, 시장이 나빠지면 CapEx압박을 상대적으로 덜 받는다. 

요약하자면
1. 전력/변전소 통제권이 높음
2. 이미 810MW를 실제 운영
3. 아직 팔지 않은 전력이 많다. (상방 열림)
4. BTC -> AI 전환으로 운영 리스크 낮다.
