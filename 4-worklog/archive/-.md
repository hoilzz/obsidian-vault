- 서브에이전트
	- 오케스트레이터(메인 클로드)가 범위가 제한된 작업을 위임할 수 있는 별도의 프로세스
		- 메인 에이전트의 컨텍스트 부담 줄여준다.
		- 특정 역할에만 집중하도록 스코프 좁힐 수 있다.
	- Skills와 궁합
		- 특정 Skill의 부분 집합만 실행할 수 있는 서브에이전트 만들어서
		- 해당 작업 자율 수행 가능
		- 필요한 경우 도구 권한 제한한 샌드박스 환경 격리 가능
```
~/.claude/agents/
  planner.md           # 기능 구현 계획 수립

  code-reviewer.md     # 코드 품질 / 보안 리뷰
  security-reviewer.md # 취약점 분석
```

- Skills, Commands
	- Skills
		- 규칙에 가까운 개념.
		- 특정 스코프와 워크플로우에 제한된 프롬프트 묶음
		- 반복해서 쓰는 작업흐름을 짧은 명령처럼 재사용 가능하게 해준다.
		- **이 작업할 때는 이 방식으로** 암묵지의 코드화
		- ex.
			- 안쓰는 코드, 정리 안된 .md 많으면
			- \/refactor-clean, \/tdd, \/e2e, \/test-coverage
	- Command 체이닝
		- Skills와 Commands는 하나의 프롬프트에서 체이닝 가능
			```
			/refactor-clean, /tdd, /e2e
			```
		- 코드 정리 -> 테스트 작성 -> E2E 실행을 한번에 할 수 있음
	- 고급활용: codemap-updator
	- 이런 Skill을 만들 수 있는데
	- 특정 체크포인트마다
	- 코드베이스의 구조 요약(codemap)을 업데이트
	- claude가 전체 코드를 다시 탐색하지 않아도 빠르게 구조 파악 가능
		- -> 컨텍스트 낭비 방지용 네비게이션 맵

```
~/.claude/skills/
  pmx-guidelines.md      # 프로젝트별 패턴
  coding-standards.md    # 언어/스타일 베스트 프랙티스
  tdd-workflow/          # README 포함 다중 파일 Skill
  security-review/       # 체크리스트 기반 보안 점검
```
