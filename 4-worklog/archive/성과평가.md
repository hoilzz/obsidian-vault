# **1. 2025년에 내가 만든 주요 성과와 기여

2025년에는 카카오페이지 글로벌 원빌드 프로젝트에서 주요 스프린트를 리딩하며, 글로벌 가입 및 인증 플로우의 설계와 전반적인 개발 조율을 담당했습니다. locale 기반의 가입·인증 흐름을 정리하고 다국어 처리 구조를 설계해 팀 내 방향성을 맞추는 데 기여했으며, 주요 결정 사항을 문서화해 협업 과정의 일관성을 확보했습니다.

프로젝트 과정에서 기획 변경과 요구사항 누락이 반복되던 문제를 개선하기 위해, 기획서를 기반으로 실패하는 e2e 테스트 시나리오를 먼저 작성하는 방식을 제안하고 도입했습니다. AI를 활용해 테스트 작성 부담을 줄였고, 이를 통해 리팩터링 및 구조 변경 시 발생하던 리그레션 이슈를 감소시키는 데 도움을 드렸습니다.

또한 디자인 시스템과 디자인 가이드를 AI 기반으로 자동화할 수 있는 figma-to-code 흐름을 정립했습니다. Code Connect와 디자인 토큰을 중심으로 프롬프트 구조를 마련하고 이를 문서화해 공유함으로써, 팀원들이 동일한 규칙과 구조 안에서 빠르게 작업을 시작할 수 있는 기반을 마련했습니다. 실제 요일연재 디자인 시스템·디자인 가이드 작업에도 적용해 초벌 개발 속도를 높였으며, 이후 반복적으로 발생하는 디자인 기반 개발 작업의 일관성을 강화했습니다.

이외에도 GraphQL → Adapter 마이그레이션 AI 작업 기여 및 API Wiki 문서를 → JSON 자동화 워크플로우 구축을 통해 코드 생성·정의 관리 과정에서 반복되는 수동 업무를 줄이는 데 기여했습니다.


## 2. [업적평가] 본인의 조직 성과 기여도 평가

2025년에는 글로벌 원빌드 프로젝트의 주요 구간을 리딩하며, 여러 기능이 동시에 진행되는 환경에서 요구사항을 정리하고 우선순위를 명확히 하며, 의사결정 내용을 문서화해 개발 과정의 안정성을 높이는 데 기여했습니다.

기획서 기반 e2e 테스트 방식을 도입한 경험은 단기간이었지만, 당시 반복되던 리그레션 문제를 줄이는 데 효과가 있었고 기획 이해 및 요구사항 누락 방지에도 도움이 됐습니다.

또한 figma-to-code 자동화 흐름을 구축하고 문서화함으로써, 팀원들이 동일한 규칙 내에서 일정한 구조로 빠르게 작업을 시작할 수 있는 기반을 마련했습니다. 디자인 기반 개발 과정의 일관성과 생산성을 높이는 데 의미가 있었다고 생각합니다. API Wiki 자동화와 Adapter 마이그레이션 역시 팀 단위 생산성과 품질 향상을 고려한 접근이었습니다.

전반적으로 맡은 역할 범위 안에서 안정적인 실행과 협업 중심의 기여를 통해 팀의 성과 달성에 도움이 됐다고 생각합니다.

## 3. [역량평가] 성과 달성 과정에서의 본인 역량 발휘 수준 평가

프로젝트 리딩 과정에서 여러 이해관계와 요구사항을 조율하며, 의사결정 사항을 명확하게 정리하고 문서화해 팀 내 공감대를 형성하는 데 주력했습니다. 글로벌 플로우처럼 복잡도가 높은 영역에서는 구조 정리와 사전 검증(R&D)을 통해 불확실성을 줄이고, 이를 공유해 개발 리스크를 낮추는 방식으로 접근했습니다.

e2e 기반 개발 방식이나 AI 자동화(figma-to-code, API JSON 파이프라인 등)는 단순한 기술 실험이 아니라 팀의 실제 업무 흐름 개선 가능성을 검증하려는 시도였습니다. 단기 생산성과 장기 구조 개선을 함께 고려하며 실행·검증·정리하는 방식을 유지하려 노력했습니다.

종합적으로 전문성, 실행력, 커뮤니케이션 역량을 균형 있게 발휘하며 맡은 범위 내에서 성과 달성에 필요한 역할을 성실히 수행했다고 생각합니다.

## **1. 올해 일하면서 가장 보람 있었던 순간**

글로벌 원빌드 프로젝트의 리딩을 맡아 팀원과 함께 설계부터 QA, 배포까지 전 과정을 마무리했을 때입니다. 특히 웹뷰 기반의 글로벌 인증 플로우를 설계하며 웹과 앱 간의 커뮤니케이션을 안정적으로 정의하고, 프로젝트의 모든 의사결정을 문서화하여 일정 안에 런칭에 성공했을 때 큰 성취감을 느꼈습니다.

## **2. 올해 목표와 성과 및 조직 기여도**

- **주요 성과:**
    - **글로벌 원빌드 리딩:** 가입/인증 퍼널 설계 및 글로벌과 국내 인증 플로우 설계, 다국어 처리 및 SEO 최적화 주도.
    - **생산성 극대화를 위한 AI 리터러시 강화:** **Figma-to-code(Code Connect 활용)** 및 **API Wiki-to-code** 자동화 워크플로우를 리서치 및 문서화를 통한 전파.
- **조직 기여도:** 
	- 문서화를 통한 팀원 및 유관부서와 커뮤니케이션: 글로벌 가입/인증과 같이 복잡도가 높은 플로우를 **시각적인 순서도와 상태도(State Diagram)로 문서화**했습니다. 이 문서는 팀 내 플로우 설계 시 **의사결정의 기준**이 되었을 뿐만 아니라, **앱 개발팀과의 커뮤니케이션에서도 핵심적인 인터페이스 정의서 역할**을 하여 하이브리드 환경의 불확실성을 완벽히 제거했습니다.
	- **AI 리터러시 강화를 통한 생산성 혁신:** 프론트엔드 팀의 제한된 자원을 극복하기 위해 AI 자동화 리서치와 실무 적용 가능성을 리서치 및 적용했습니다.
		- **Figma-to-code:** Code Connect 기반의 자동화를 구축하여 디자인 가이드와 개발 간의 간극을 좁히고, 반복적인 UI 작업 시간을 대폭 단축했습니다.
		- **API Wiki-to-code:** 수동으로 관리되던 API 정의를 JSON 기반 자동화 워크플로우로 전환하여, 데이터 정합성 유지와 코드 생성 과정의 휴먼 에러를 원천 차단했습니다.

### **3. [도전/시도] 기존 방식을 넘어 새롭게 시도했거나 도전해 본 것은? 그 결과와 느낀 점은?**

- **도전 내용:** **'팀 생산성의 구조적 개선'**을 위해 AI 리터러시 강화와 실무 자동화를 시도했습니다. 특히 **Figma-to-code(Code Connect 활용)** 및 **API Wiki-to-JSON 자동화** 파이프라인을 구축하여 리서치부터 가이드 문서화까지 주도했습니다. 
- 또한, 복잡도가 높은 글로벌 인증 플로우를 **상세 시퀀스/플로우 문서로 시각화**하여 웹-앱 간의 기술적 합의를 이끌어내는 새로운 소통 방식을 도입했습니다.
    
- **결과:** 
	- 반복적인 UI 초벌 작업에서 발생하는 휴먼 에러를 어느정도 줄였습니다. 
	- 또한 명확한 플로우 문서를 통해 앱 개발팀과의 커뮤니케이션 비용을 최소화 했습니다.
	    
- **느낀 점:** 팀 리딩시 단순히 업무를 분배하는 것이 아니라, 팀원들이 겪는 기술적인 병목이나 문제를 먼저 해결하고, 누구나 동일한 구조로 작업할 수 있도록 시스템을 만드는 것이라고 느꼈습니다.

### **4. 목표 대비 기대에 미치지 못한 것은 무엇인가요? 그 이유는?**

- **내용:** 리서치와 문서화를 통해 Figma-to-code 자동화의 가능성을 입증했으나, 개발, 디자인 각각의 디자인시스템이 약간 다른점이 있어 100% 자동화는 아직 어려울 것 같습니다.
    
- **이유:** **이유:** 완전한 자동화를 위해서는 단순히 툴을 도입하는 것을 넘어, LLM이 시스템을 정확히 이해하고 코드를 생성할 수 있도록 **디자인 토큰과 컴포넌트 구조를 재정립하는 선행 작업**이 필요하기 때문입니다. 이는 디자인팀은 물론 iOS, Android 등 타 클라이언트 팀과의 전사적인 규격 합의가 필수적인 영역입니다. 이를 보완하기 위해 내년에는 **유관 부서와 긴밀히 협업하여 디자인 시스템의 구조적 최적화**를 이뤄내고 싶습니다. 조직 전반의 **AI 리터러시를 함께 높여**, 전사적으로 한정된 자원 내에서 개발 효율을 극대화할 수 있는 환경을 만드는 것을 내년의 주요 과제로 삼고 싶습니다.

### **5. [역량평가] 역량 측면의 성장과 보완이 필요한 역량은?**

- **성장:** 프로젝트 전체 라이프사이클(설계~QA~배포)을 총괄하는 리딩 역량이 어느정도 성장했다고 생각합니다. 특히 웹뷰와 웹 환경의 유저 인증 플로우 아키텍처 설계 역량과, 팀 생산성을 극대화할 수 있는 **AI 리터러시 기반의 자동화 기획 역량**을 갖추게 되었습니다. 또한, 복잡한 로직을 시각화하여 유관 부서와 합의를 끌어내는 **기술 커뮤니케이션 역량**도 강화되었습니다.
    
- **보완:** 올해 리서치한 자동화 프로세스들이 단순히 우리 팀의 효율을 높이는 데 그치지 않고, **전사적인 생산성 향상으로 이어질 수 있도록 시스템화하는 역량**을 보완하고 싶습니다. 타 부서와의 협업 구조를 정교하게 설계하여 내가 구축한 자동화 모델이 조직 전체의 표준으로 정착될 수 있게 유도하고, 이를 통해 모든 팀이 한정된 자원 속에서 더 높은 가치를 창출할 수 있도록 더욱 보완하겠습니다.

### **6. 업무에 도움을 준 동료가 있다면? 구체적으로 어떤 도움을 받았나요?**

- **내용:** 글로벌 원빌드라는 긴박하고 복잡한 프로젝트 일정 속에서, 주도적으로 개발과 QA에 매진한 팀원들 이라고 생각합니다. 또한, n8n 활용이나 AI 관련 리서치 등 팀의 기술적 스펙트럼을 넓혀준 동료들이 있었기에 우리 팀이 **'안정적인 서비스 운영'과 '기술적 혁신'**이라는 두가지 모두 성취할 수 있었다고 생각합니다.
    
- **기여 포인트:** 동료들이 안심하고 새로운 시도를 할 수 있도록 스프린트 팀이 중심을 잘 잡아주고, 반대로 동료들의 혁신적인 결과물을 다시 실무 자동화에 녹여내는 **선순환의 시너지**를 경험할 수 있었던 소중한 시간이었습니다.


### **7. 회사생활이나 업무에 있어 애로사항이나 지원사항이 있다면 무엇인가요?**

- **내용:**  따라서 올해 리서치한 **AI 기반 자동화(Figma-to-Code, API-to-Code 등)를 단순한 실험을 넘어 실무 프로세스에 완전히 이식하는 것**이 제 개인적인 성장과 팀의 주요과제라고 생각합니다.
    
- **지원사항:** 이를 위해 디자인팀, 클라이언트팀(iOS, aOS)과 함께 논의할 필요가 있다고 생각합니다. 부서 간 협업할 수 있는 시간이 주어진다면, 우리 팀의 자동화 성과가 조직 전체의 생산성을 높이는 좋은 사례가 될 수 있다고 생각합니다.
