### pnpm

- .pnpm-store(글로벌 저장소)에 실제 파일이 저장된다. 그래서 한번 다운로드 되면 다음부터 재다운로드 없다
- node_modules는 하드 링크된 가상 구조
	- 즉, 용량 절약 설치도 빠르다

pnpm 은 캐시 히트 여부에 따라 pnpm install 실행 여부를 결정하지 않는다. 무조건 pnpm install을 한다. pnpm은 npm,yarn과 다르게 ~/.pnpm-store를 기준으로 설치한다. 즉, 캐시가 있으면 실제 node_modules에 거의 링크만 복사한다. 그래서 pnpm i 가 실행되더라도 실제 다운로드 없이 빠르게 끝난다. 

pnpm install은 내부적으로 node_modules 정합성 검사도 같이한다. lockfile과 불일치하는 모듈 등을 같이 확인 하기 때문에 한 번 실행하는 것이 안정적이다. 

> npm i 는 진짜로 느리기 때문에 캐시 히트시 node_modules를 복사 하고 설치를 건너 뛰어도 된다. pnpm은 구조적으로 store를 캐싱하면 install 부담이 없다.
