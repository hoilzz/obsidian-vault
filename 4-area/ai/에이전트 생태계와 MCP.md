에이전트가 툴을 어떻게 협력하고 발전할 수 있는지, MCP가 이들을 표준화하고 어떻게 더 효율적인 시스템을 구축할 수 있게 만드는지 알아보자.

#### 1. 에이전트 성능 결정 요인
- **챗봇**의 성능은 LLM (대형 언어 모델)이 결정하듯, **에이전트**의 성능은 **툴(API)**에 달려 있습니다.
- 멀티 펑션: 하나의 툴이 여러 작업 연속적 수행
- 멀티 툴: 여러 툴이 협력하여 하나의 에이전트가 다양한 작업 처리

#### 2. 기존의 한계
- ChatGPT와 같은 시스템은 각 에이전트가 고립되어 상호작용 어려움.

#### 3. MCP의 등장
- [[MCP]]는 에이전트와 LLM, 그리고 툴 간의 상호작용을 __표준화__ , 툴의 기능과 데이터를 LLM이 이해할 수 있는 형태로 변환하는 중간 계층 역할 (표준화가 핵심)

#### 4. 에이전트 생태계의 미래
- MCP는 에이전트 생태계의 표준 프로토콜로 성장할 가능성이 큼. HTTP가 웹에서 표준 프로토콜이 된 것 처럼
- MCP는 에이전트가 툴을 선택하고, 기능을 탐색하고 상호작용할 수 있는 지능형 인터페이스로 작용하게 될 거임
