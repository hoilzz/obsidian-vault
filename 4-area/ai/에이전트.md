### 기존 [[LLM]] 의 역할
주로 텍스트를 처리하고 이에 맞는 반응을 생성한다. 챗봇, 문서요약, 번역기 등
기존 LLM은 자체적으로 외부 툴을 사용하지 않는다. 예컨대, 날씨 확인 혹은 메일 보내기 작업을 수행할 수 없다. __주어진 텍스트만을 기반으로 반응한다.__

### 역할

__구체적인 작업을 수행하는 AI 시스템이다.__ 여러 툴 (API)을 활용하여 실제 작업을 실행할 수 있다. 에이전트는 LLM 기반이지만, 툴을 사용하여 __실제 동작을 수행하는 점__ 에서 차이가 있다.에이전트는 LLM을 이해 + 실제 작업을 수행한다. 예컨대
- 툴 사용: 캘린더 앱을 사용해 일정 확인, 메일 API를 호출해 이메일 보내기 등의 작업 가능
- 작업의 흐름: 복잡한 워크플로 처리 가능. 회의 일정 확인, 참석자 정보 바탕으로 요약 작성, 그 결과를 이메일로 발송하는 작업을 하나의 에이전트가 처리 가능.
