- 에러는 코드와 함께 보내줘야 적절한 처리 가능
- fetchApi의 적절한 추상화
	- axios.create 메소드가 비슷한 패턴을 사용한다.
- API 응답 에러처리
- 클래스 네임 처리 ㄴㄴ
	- 클래스 네임이 바뀌면 여기도 바뀌어야 하는데 개발자가 다 챙기기 어려움 휴먼에러 발생 가능성 높음
- App.js 코드가 복잡해진다는거 인지함
	- 사실 제일 이상적인건 상태가 변경되면 알아서 리렌더링
	- 컴포는트에 setState가 아닌 값을 전달한다고 해보자.
	- 상태는 App한곳에서 다뤄지고 여기서 값이 변경
	- 변경된 값을 파라미터로 받는 컴포넌트가 있다고 했을 때 알아서 리렌더링 + 실제 변경된 값이 있을 때 (참조값 변경, primitive 값 변경)만 알아서 리렌더링
	- 이걸 직접 구현해볼 생각을 해도 좋다. 이런걸 직접 매번 구현하기 귀찮으니까 react, svelte 이런 얘들이 존재. 
	- 일단 지금 상황에서 빠르게 해결가능한 방법은
	- 각 컴포넌트에서 childrens로 컴포넌트 인스턴스를 들고 있고
	- 각 컴포넌트의 setState가 일어날 때 마다 childrens를 리렌더링 할 껀데
	- chidlrens가 받는 파라미터들을 메모이징 해두고 기존 메모이징 값과 새롭게 setState된 상태를 deep compare하여 바뀐게 하나라도 있다면 리렌더 없으면 안함.
