- RESTful하게 작성할 것인지?
    - RESTful이 뭘까요
    - Representational State Transfer. 상태를 Representation했다는게 무슨 의미일까요. 무슨 상태?
    - API를 통해서 우리한테 오는 저 JSON 데이터. 저게 원본임? 원본은 DB에 있다. API 응답에 들어있는 JSON을 그저 현재 DB에 들어있는 원본 데이터를 “표현”하고 있는 것일 뿐.
    - 즉, RESTful한 API는 “뭘 표현할 것이냐”에 집중해서 설계되어야 한다. 원론적으로는 완전히 모델 관점임.
    - 물론 인생 살다보면 이렇게 아름답게만 설계하기는 어렵긴 함. 다만 이렇게 하는게 좋지 않다는 걸 인지한 상태에서 뭉개고 가는 것과 아무것도 모르고 뭉개버리는 것은 천지차이. 모르고 뭉개버린다는 의미는 나중에 개선하겠다는 생각도 안 들거란 얘기임.
