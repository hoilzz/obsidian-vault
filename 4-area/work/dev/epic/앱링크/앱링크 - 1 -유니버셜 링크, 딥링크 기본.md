### 유니버셜 링크, 딥링크

https://juneyr.dev/universal-link

#### 유니버셜 링크
- iOS에서만 작동하는 딥링크의 한 종류
- 딥링크는 앱과 앱을 오가는 쉽게 해줌.
	- 웹 보다가 특정화면만 유저가 앱으로 봤으면 하는 경우 딥링크 작동.
	- 이 때 앱을 깔아두었으면 바로 해당화면 이동
	- 딥링크의 형태는 https:// , app://open.my.app 둘다 지원한다.
![[스크린샷 2023-04-12 오전 10.29.43.png]]

> 새로운 모바일 게임 출시 사전등록함.
> 출시 당일에 링크 하나가 오는데 보통 카톡으로 이 링크를 누를거다. 
> 링크 누르면 페이지 하나 떴다가 스토어로 이동 (앱 미설치라)
> 혹시나 앱이 설치되어있다면 링크 누를 경우 앱으로 이동 (순간적으로 딥링크 사용)

### 딥링크의 두가지 종류

1. default deep link: 유저가 앱을 깔았을 때만 원하는 화면 이동. __앱 없으면 에러 띄움__
	1. 유니버셜 링크(ios)
	2. 앱링크 (aos)
	- 앱이 없으면 에러를 띄우는데, 개발자가 앱스토어로 이동할 건지 웹에서 남을건지 결정해서 구현 즉, 미리 앱을 깐 유저들을 데려오고 싶은 목적일 때 유용

1. deferred deep link: 앱 깔려있지 않은 유저 App, play store 데려가고 설치된 뒤에 원래 보고싶었던 페이지를 띄우게 함

> 앱 스킴 : app://open.my.app?params=option
> http 링크 https://myhome.com

## 앱스킴과 http 링크

- 앱 스킴 (App Scheme, URL Scheme)
	- `tapas://명령어?파라미터=옵션&버전`
	- 이 링크가 걸린 버튼을 클릭하면 모바일에서는
		- 위 scheme이 시스템에 전달
		- 시스템에서는 url scheme을 보고 실행가능한 앱이 있나 확인
			- 가능하면 앱을 실행시키면서 scheme을 전달.
			- 불가능하면
				- iOS: 해석불가능한 Scheme이므로 오류 팝업 or 페이지가 뜸.
					- 이 경우 appstore 링크로 이동시킬 수 있음
				- aos: intent scheme을 사용하여 자동으로 구글플레이 이동
					- intent가 없으면 iOS처럼 예외처리 가능
- 앱과 웹을 연결시 위와 같이 scheme만을 사용했음. 
- __그러나 http url 형태를 사용하면 사용자의 OS버전에 상관없이 원하는 앱 화면으로 이동 시킬 수 있음__
	- 즉 일관성있으며 http url은 unique하다는 장점 있음.
	- app scheme은 사실 앱 들 간 겹칠 수도 있기 때문이다.
