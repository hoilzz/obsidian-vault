QNA

- getStaticProps 왜쓰나
	- 위 함수에서 리턴된 props를 이용하여 빌드 타임에 페이지를 pre-render 한다(Static Site Generation)
- getStaticProps는 다음 상황에서는 꼭 사용되야하는데..
	- 빌드 타임에 이용할 수 있는 렌더링에 꼭 필요한 데이터
	- 데이터가 헤드리스 CMS에서 오는 경우
	- SEO 떄문에 페이지가 프리렌더 되야아하는 경우, 그리고 굉장히 빨라야하는 경우
 		- getStaticProps는 HTML과 json file을 만드는데 둘다 CDN에 캐싱될 수 있다. 이것은 성능상 이점이 있다. 
 		- 데이터는 공개적으로 캐싱될 수 있다. (사용자별 ㄴㄴ) 이 조건은 미들웨어에서 path를 재작성 하여  특정 상황에서 바이패스될 수 있다.

- getStaticProps가 실행되는 시점
	- 서버에서만 실행된다. 클라에서 절대 실행 안된다. getStaticProps 내부에서 작성된 코드를 검증할 수 있다.
	- revalidate를 사용하면 background에서 on-demand로 실행된다.
	- `Incremental Static Regeneration` 과 결합하면 getStaticProps는 오래된 페이지의 유효성을 다시 확인하고 새로운 페이지를 제공하는 동안 백그라운드에서 실행된다.

- getStaticProps는 incoming request에 접근 못함. **(query parameter!!)** 왜냐하면 static HTML로 생성하기 때문이다.
- 만약 페이지 request에 접근할 필요가 있다면 getStaticProps에 Middleware를 사용하는 걸 고려해라.
