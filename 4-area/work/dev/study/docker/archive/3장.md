### 3-2
- Dockerfile: 애프리케이션을 패키징하기 위한 스크립트
	- 일련의 인스트럭션으로 구성돼있다. 
		- 자주쓰이는 인스트럭션
		- FROM
			- 모든 이미지는 다른 이미지에서 출발
			- `FROM diamol/node`
				- diamol/node 이미지를 시작으로 하는데, 웹 어플리케이션을 실행할 수 있는 node.js가 설치되어있음
			- ENV
				- 환경 변수 값 지정
			- WORKDIR
				- 작업 디렉터리를 만든다.
			- CMD
				- 명령어 실행
### 3-3 컨테이너 이미지 빌드
- 이미지를 빌드하려면 Dockerfile 스크립트 외에도 필요한 게 있음
- 이미지의 이름, 패키징에 필요한 파일의 경로 지정
- `--tag` : 의 인자값은 이미지의 이름, 그 뒤는 이미지에 포함시킬 파일이 위치한 경로
	- `docker image build --tag web-ping .` : 이미지 이름은 web-ping이고 현재 작업 디렉터리를 이미지에 포함시켜라



### 3-4 도커 이미지와 이미지 레이어 이해하기
- 도커 이미지는 패키징에 포함시킨 모든 파일이 들어감
	- 이미지 레이어가 모인 논리적 대상
	- 레이어는 도커 엔진의 캐시에 물리적으로 저장된 파일
	- 즉, node 앱이 실행되는 컨테이너 여러개 실행한다면 이 컨테이너는 모두 node 런타임이 들어있는 이미지 레이어 공유
- 이 파일은 나중에 컨테이너의 파일 시스템을 형성
