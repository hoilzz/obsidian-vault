## 요약

- 도커 이미지는 도커 컨테이너를 생성하는데 사용되는 템플릿.
	- 애플리케이션 코드, 라이브러리, 종속성, 설정 파일등을 포함한다. 이를 기반으로 도커 컨테이너가 실행됨.
	- 계층 구조
		- 이미지 빌드 과정에서 Dockerfile의 명령어에 의해 생성됨. (story-fe의 도커파일을 참고)
		- 프로스스가 실행되는 파일들의 집합. 
			- 이 프로세스는 환경(파일)을 변경할 수 있고 이 환경을 저장하여 새로운 이미지를 만듬.
	- 이식성
		- 도커 이미지는 개발, 테스트, 프로덕션 환경 간 이식성이 뛰어나다.
			- 이미지 한 번 빌드시 어디서든 동일한 환경 재현 가능
	- 버전 관리 가능
		- nginx:latest, nginx:1.19.3

### 이미지

- 도커는 레이어드 파일 시스템 기반
- 이미지는 프로세스가 실행되는 파일들의 집합(환경)
	- 리눅스는 여러개의 파일이 있는데 전체 파일이 다필요. 리눅스 전체파일을 압축해놓은 그런 느낌
- 프로세스는 환경(파일)을 변경할 수 있고 이 환경을 저장하여 새로운 이미지를 만듬
	- 차곡차곡 쌓인 이미지를 합쳐 또다른 이미지를 만듬.
