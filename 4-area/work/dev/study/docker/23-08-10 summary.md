- 컨테이너란?
	- __격리__ 된 환경에서 작동하는 프로세스가 동작하는 기술
	- 그래서 컨테이너 떴나요? 이런말을 하는 거 같다.
	- 컨테이너라는 표준으로 서버를 배포하므로 모든 서비스들의 배포과정 동일해진다.
		- 기존에는 OS를 가상화.(성능 이슈) 도커는 프로세스 격리
			- 단순 프로세스 격리라 빠르게 동작 + CPU나 메모리 프로세스가 필요한 만큼만 사용
- 이미지란?
	- 컨테이너를 실행하기 위한 파일과 설정값을 포함
		- 상태값을 가지지 않고 이뮤터블함
		- 이미지는 컨테이너를 실행하기 위한 모든 정보와 파일을 가진다 = 의존성 파일 컴파일 및 이것저것 설치할 필요 전혀 없음
	- 이미지에서 컨테이너를 생성하기 때문에 반드시 이미지 만드는 과정이 필요
		- **컨테이너는 이미지를 실행한 상태**
	- 보통 Dockerfile로 이미지를 만든다. 
		- 그래서 빌드 서버에서 이미지를 만들면 해당 이미지를 이미지 저장소에 저장, 운영서버에서 이미지를 불러온다. 
- 도커란?
	- 컨테이너 기반의 가상화 플랫폼
	- 왜 핫할까?
		- 레이어 저장방식 
			- 기존 이미지에 파일 하나 추가할 때 전체를 다시 다운받으면 비효율이라 레이어 개념 사용하여 새로 추가된 레이어안 다시 받음.
		- 이미지 경로 
			- url방식 + 태그로 관리
		- 도커 허브 무료
			- 큰 용량의 이미지들을 무료로 도커허브에서 관리 가능 

- Dockerfile
	- 도커파일을 읽어서 이미지를 만들 수 있음.

- 도커 컴포즈
	-  yaml 파일로 관리되고, 멀티 컨테이너 어플리케이션을 정의하고 공유하는데 도움이 됨.
