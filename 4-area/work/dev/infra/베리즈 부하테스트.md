[참고](https://kakaoent.atlassian.net/wiki/spaces/KAKAOENTFE/pages/3733651713/berriz+FE)
목표는 400 TPS, 0.2초 레이턴시 
단일 Pod 기준으로 최적의 CPU, memory를 찾고 HPA 설정 완료

테스트 방향
- 실제 상황과 유사하게 API까지 연동
- 가장 컨텐츠가 많은 페이지 기준으로 결과 도출
	- pod 한대가 max TPS, latency를 받을 수 있는 적정 CPU, memory 찾자.
		- FE의 경우 DB가 있거나 write, update와 같은 작업이 있는 것이 아니기 때문에 pod 한대당 tps 기준으로 선형 계산하면 목표치를 달성할 수 있을 거라 판단함

노드 스펙
- n2-standard-8 (8vCpus, 32GB) 노드 3개
	- 물리 코어는 4코어고 하이퍼스레딩으로 8개의 vCPU로 동작 가능
- 노드 풀 2개
