무중단 배포를 목표로 하는 배포 전략 중 하나다. 2개의 환경 (블루, 그린)을 운영하면서 새로운 배포를 안전하게 수행하는 방식

- 블루: 현재 Production 중인 버전
- 그린: 새로운 버전이 배포될 환경

새로운 버전을 Green 환경에 배포한 후, 트래픽을 Green으로 스위칭하면 배포 완료

### 배포과정

1. 블루에서 서비스 환경 운영
2. 그린 환경에 새로운 버전 배포 및 테스트
3. 그린 환경 테스트 완료 후 트래픽 스위칭
4. 이전 블루 환경 유지 또는 제거

#### 장점
- 무중단 배포: 트래픽 점진적 전환
- 롤백 빠름: 문제 발생시 이전(블루) 환경으로 트래픽 다시 전환
- 배포 전 그린으로 미리 테스트 가능
	- 근데 우리 서비스는 외부 트래픽 없이 특정 쿠키값이 있을 때 그린을 바라본거 같기도..?
#### 단점
- 블루,그린 동시 운영하여 리소스 비용 증가
