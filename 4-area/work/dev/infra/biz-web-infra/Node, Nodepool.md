## 요약
- Node: 클러스터의 기본 컴퓨팅 유닛. 애플리케이션이 실제로 실행되는 물리적 또는 가상 머신
- Node Pool: 동일한 특성을 가진 노드의 그룹. 특정 리소스 요구사항을 충족하기 위해 별도 관리되고 스케일링될 수 있음. 

### Node
- 노드는 클러스터를 구성하는 기본 컴퓨팅 유닛. 걍 실제 물리 서버 or VM이라 생각하자. 
- 주요 구성 요소
	- kubelet: 쿠버네티스 API 서버와 통신하며, 각 컨테이너가 잘 떠있는지 확인
	- 컨테이너 런타임: 컨테이너를 실행하고 관리 (Docker)
	- kube-proxy: 네트워킹 담당, 쿠버네티스 서비스의 네트워크 규칙을 유지. 
- 종류
	- 마스터노드: 클러스터의 control plane, 클러스터 상태 관리 및 작업 스케줄링
	- 워커노드: 실제 어플리케이션 워크로드가 실행됨.

## Node pool
- 노드그룹임
- 주요 특징
	- CPU 집약적인 작업을 위해 고성능 CPU가 있는 노드풀 만들거나 메모리 집약적인 작업을 위해 메모리 많은 노드풀 생성가능
	- 스케일링 가능. 
	- 구성 유사성: 노드풀 내의 모든 노드는 유사한 구성을 가진다.
