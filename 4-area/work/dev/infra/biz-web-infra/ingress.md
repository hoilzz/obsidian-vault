인그레스는 클러스터 외부 -> 내부 서비스로 HTTP, HTTPS 경로를 노출한다


---
클러스터 내의 서비스에 대한 외부 접근을 관리하는 API 오브젝트. 일반적으로 HTTP를 관리
부하 분산, SSL 종료, 명칭 기반의 가상 호스팅 제공

- 인그레스는 클러스터 외부 -> 내부 _서비스_ 로 HTTP, HTTPS 경로를 노출
- 트래픽 라우팅은 인그레스 리소스에 정의된 규칙에 의해 컨트롤됨.
- 인그레스 컨트롤러는 로드밸런서를 사용하여 인그레스를 수행할 책임이 있다. 
	- 트래픽 처리 등 에지라우터 추가 프런트엔드 구성
- 임의의 포트 또는 프로토콜 노출 안함
- HTTP, HTTPS 이외에 서비스를 인터넷에 노출하려면 ServiceType NoderPort, LoadBalancer 유형의 서비스 사용

## 전제 조건

- [[인그레스 컨트롤러]] 가 있어야 인그레스를 충족할 수 있다. 인그레스 리소스만 생성한다면 효과 없음.
- [Ingress-nginx](https://kubernetes.github.io/ingress-nginx/deploy/) 와 같은 인그레스 컨트롤러를 배포해야할 수도 있음. 여러 인그레스 컨트롤러 중 선택.o
